## 3. 前端技術詳解

### 3.1 Vue 3 Composition API

所有元件均採用 `<script setup>` 語法：

```vue
<script setup>
import { ref, computed, onMounted } from 'vue'

// 響應式資料
const count = ref(0)

// 計算屬性
const doubled = computed(() => count.value * 2)

// 生命週期
onMounted(() => { /* 初始化 */ })
</script>
```

**使用的 Composition API 函式：**

| 函式 | 用途 |
|---|---|
| `ref()` | 基本型別響應式資料 |
| `reactive()` | 物件型別響應式資料（表單） |
| `computed()` | 計算屬性（篩選、加總） |
| `watch()` | 監聽資料變化（如 store → form 同步） |
| `onMounted()` | 頁面載入時呼叫 API |

### 3.2 Tailwind CSS v4

採用 **Vite Plugin** 方式整合（非 PostCSS），不需要 `tailwind.config.js`：

```js
// vite.config.js
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [vue(), tailwindcss()],
})
```

```css
/* src/style.css */
@import "tailwindcss";
```

**主要用法模式：**

```html
<!-- 響應式斷點 -->
<div class="h-[560px] md:h-[680px]">

<!-- 群組互動 (Feature Card hover) -->
<div class="group">
  <p class="max-h-0 group-hover:max-h-32 transition-all duration-500">

<!-- 任意值 -->
<div class="bg-black/70">   <!-- rgba(0,0,0,0.7) -->
```

### 3.3 Swiper.js 輪播

首頁 Hero 輪播使用 Swiper v12，啟用 **EffectFade（淡入淡出）** 效果：

```vue
<script setup>
import { Swiper, SwiperSlide } from 'swiper/vue'
import { Autoplay, Pagination, Navigation, EffectFade } from 'swiper/modules'
import 'swiper/css'
import 'swiper/css/pagination'
import 'swiper/css/navigation'
import 'swiper/css/effect-fade'

const modules = [Autoplay, Pagination, Navigation, EffectFade]
</script>

<template>
  <Swiper
    :modules="modules"
    effect="fade"
    :autoplay="{ delay: 4500, disableOnInteraction: false }"
    :pagination="{ clickable: true }"
    :navigation="true"
    :loop="slides.length > 1"
  >
    <SwiperSlide v-for="slide in slides" :key="slide.id">
      ...
    </SwiperSlide>
  </Swiper>
</template>
```

**Swiper CSS 變數客製化（`<style scoped>`）：**

```css
.hero-swiper {
  --swiper-pagination-color: #fff;
  --swiper-navigation-color: #fff;
  --swiper-navigation-size: 24px;
}
.hero-swiper :deep(.swiper-button-prev),
.hero-swiper :deep(.swiper-button-next) {
  background: rgba(0, 0, 0, 0.35);
  border-radius: 50%;
  backdrop-filter: blur(4px);
}
```

## 5. 路由設計

### 5.1 Hash History Mode

```js
// src/router/index.js
import { createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  // URL 格式：https://example.com/japanese-restaurant/#/menu
})
```

**選用 Hash Mode 的原因：**
GitHub Pages 為靜態檔案伺服器，無法設定 HTTP 重寫規則。Hash Mode 所有路徑變化均在前端處理，不會觸發伺服器 404。

### 5.2 路由結構

```
/ (FrontLayout)
├── /              → HomeView      前台首頁
├── /menu          → MenuView      菜單列表
├── /product/:id   → ProductView   商品詳情
├── /cart          → CartView      購物車 / 結帳
└── /order/:id     → OrderView     訂單確認

/admin             → LoginView     後台登入（無 Layout）

/admin (AdminLayout) [requiresAuth: true]
├── /admin/products  → ProductsView  產品管理
├── /admin/orders    → OrdersView    訂單管理
├── /admin/coupons   → CouponsView   優惠券管理
├── /admin/banners   → BannersView   輪播圖管理
├── /admin/features  → FeaturesView  特色區塊
├── /admin/cta       → CtaView       CTA 區塊
└── /admin/footer    → FooterView    Footer 編輯
```

### 5.3 路由守衛

```js
router.beforeEach(async (to) => {
  if (to.meta.requiresAuth) {
    const auth = useAuthStore()
    const ok = await auth.verify()       // 每次導航都驗證 Token
    if (!ok) return '/admin'             // 驗證失敗 → 重導至登入頁
  }
})
```

### 5.4 路由懶載入

所有 View 元件均採用動態 `import()` 實現程式碼分割：

```js
component: () => import('@/views/front/HomeView.vue')
```

效果：首次載入只下載首頁程式碼，其他頁面切換時才下載。

### 6.2 分頁機制

產品、訂單、優惠券列表使用 HexSchool API 的分頁機制（每頁 10 筆）：

```js
// API 回傳的 pagination 物件
{
  total_pages: 3,
  current_page: 1,
  has_pre: false,
  has_next: true,
  category: ""
}

// 換頁
async function fetchOrders(page = 1) {
  const res = await getAdminOrders(page)
  orders.value = res.data.orders
  pagination.value = res.data.pagination
  currentPage.value = page
}
```

### 6.3 訂單類型識別

API 訂單資料本身沒有「內用/外帶」欄位，系統透過訂單 `message` 欄位的前綴判斷：

```js
function getOrderType(order) {
  const msg = order.message || ''
  const name = order.user?.name || ''

  if (msg.startsWith('【內用】') || name.startsWith('桌號')) {
    // 從 message 用正規表達式取得桌號
    const match = msg.match(/桌號：(.+?)(?:　|$)/)
    return { type: 'dine-in', table: match?.[1] || '' }
  }
  return { type: 'takeout', table: '' }
}
```

回傳的 `type` 用來決定顯示 `t('admin.common.dineIn')` 或 `t('admin.common.takeout')`，不再內含硬編碼中文字串。

**內用結帳時儲存格式：**
```
user.name  = "桌號 A3"
message    = "【內用】桌號：A3　2026/03/26 12:30"
```

### 6.4 後台 LocalStorage 編輯器模式

`banners`、`features`、`cta`、`footer` 的資料不走 API，直接修改 Pinia store，由 `pinia-plugin-persistedstate` 自動寫入 `localStorage`。

```js
// CtaView.vue — 表單與 store 雙向同步
const { t } = useI18n()
const form = reactive({ ...store.data })

// 監聽 store 外部變更同步回表單（如重置預設值）
watch(() => store.data, val => Object.assign(form, val), { deep: true })

function handleSave() {
  store.update({ ...form })          // 寫入 store → localStorage
  toast.success(t('toast.ctaSaved')) // 翻譯後的通知文字
}
```

### 6.5 圖片預覽機制 (Image Preview)

`BannersView`、`FeaturesView`、`CtaView` 編輯區內含圖片預覽面板。預覽圖片以獨立的 `ref` 管理，**不直接綁定表單輸入**，避免邊打字邊發出大量圖片請求。

#### 預覽更新時機

| 動作 | 是否更新預覽 |
|---|---|
| 在 URL 輸入框打字 | ❌ 不更新 |
| 點擊「更新預覽」按鈕 | ✅ 立即更新 |
| 點選快速圖片縮圖 | ✅ 立即更新 |
| 開啟 Modal（編輯模式） | ✅ 自動填入現有圖片 |
| 開啟 Modal（新增模式） | ✅ 清空預覽 |

#### 各 View 預覽 ref 對照

| View | 預覽 ref | 更新按鈕觸發 | 快速選圖觸發 |
|---|---|---|---|
| `BannersView` | `previewImage` | `previewImage = form.image` | `form.image = p; previewImage = p` |
| `FeaturesView` | `previewSrc` | `previewSrc = form.image` | `form.image = p; previewSrc = p` |
| `CtaView` | `previewBg` | `previewBg = form.bgImage` | `form.bgImage = p; previewBg = p` |

#### URL 輸入框 + 更新按鈕結構（BannersView 為例）

```vue
<div class="flex gap-2">
  <input
    v-model="form.image"
    placeholder="https://images.unsplash.com/..."
    class="flex-1 h-9 ..."
  />
  <button
    type="button"
    @click="previewImage = form.image"
    class="h-9 px-3 bg-gray-100 hover:bg-gray-200 border border-gray-300 rounded-md text-xs ..."
  >{{ t('admin.common.updatePreview') }}</button>
</div>
```

#### CtaView 特殊說明

`CtaView` 的 `form` 使用 `reactive`（非 `ref`），`previewBg` 在 `store.data` watcher 及 `handleReset` 中同步更新：

```js
const form = reactive({ ...store.data })
const previewBg = ref(form.bgImage)
watch(() => store.data, val => {
  Object.assign(form, val)
  previewBg.value = val.bgImage
}, { deep: true })
```

#### 快速選用清單 — 自訂網址新增列

快速選用縮圖下方新增一列輸入欄位，讓管理員在執行期間擴充縮圖清單，無需修改程式碼。

`presets` 已從靜態 `const []` 改為響應式 `ref([...])`，並加入 `newPresetUrl` 與 `addPreset()`：

```js
const presets = ref([ /* 預設圖片 URL 陣列 */ ])
const newPresetUrl = ref('')

function addPreset() {
  const url = newPresetUrl.value.trim()
  if (!url || presets.value.includes(url)) return  // 空白或重複不新增
  presets.value.push(url)
  newPresetUrl.value = ''
}
```

UI 結構（縮圖格下方）：

```vue
<div class="flex gap-2 mt-2">
  <input
    v-model="newPresetUrl"
    @keyup.enter="addPreset"
    placeholder="https://images.unsplash.com/..."
    class="flex-1 h-8 border border-gray-300 rounded-md px-3 text-xs ..."
  />
  <button type="button" @click="addPreset" class="h-8 px-3 bg-red-700 ... text-xs ...">
    {{ t('admin.common.addPreset') }}
  </button>
</div>
```

| 行為 | 結果 |
|---|---|
| 貼入 URL → 點「新增」或按 Enter | 縮圖加入格線末尾，可立即點選 |
| 空白或重複 URL | 忽略，不新增 |
| 頁面重新整理 | 恢復初始預設清單（非持久化） |

### 6.7 Modal 元件 — 捲動修正

**問題：** `BannersView`、`ProductsView` 等含圖片選取器的 Modal，內容過長時 footer 儲存按鈕被推出可視範圍。

**解法：** `src/components/ui/Modal.vue` 加入高度限制與獨立捲動區（commit `569646c`）：

```vue
<!-- 修改前 -->

  <div class="flex ... p-5 border-b">...標題...</div>
  <div class="p-5">...內容（可能極長）...</div>

</div>

<!-- 修改後 -->

  <div class="flex ... p-5 border-b shrink-0">...標題（固定）...</div>
```

| 類別 | 說明 |
|---|---|

### 7.2 Request Interceptor — 自動附加 Token

每個請求發出前，攔截器讀取 Cookie 中的 `hexToken` 並加入 Authorization Header：

```js
    /(?:(?:^|.*;\s*)hexToken\s*=\s*([^;]*).*$)|^.*$/, '$1'
  )
  if (token) {
    config.headers.Authorization = token
  }
  return config
})
```

## 8. 資料儲存策略

系統資料分為三種儲存方式：

```
┌─────────────────┬──────────────────────────────────────────────┐
│  儲存位置        │  資料內容                                      │
├─────────────────┼──────────────────────────────────────────────┤
│  HexSchool API  │  商品、訂單、優惠券、購物車                      │

│  localStorage   │  輪播圖、特色區塊、CTA、Footer（含社群連結）      │

├─────────────────┼──────────────────────────────────────────────┤
│  Cookie         │  hexToken（後台 JWT 認證 Token）               │
│  (瀏覽器 Cookie) │  → 有到期時間，登出時主動清除                    │
└─────────────────┴──────────────────────────────────────────────┘
```

## 13. 多國語系 (I18n)

### 13.1 技術選型

採用 **vue-i18n v9**（Vue 3 官方推薦 I18n 套件），以 `legacy: false` 啟用 **Composition API 模式**：

```js
// src/i18n/index.js
import { createI18n } from 'vue-i18n'
import zhTW from '@/locales/zh-TW'
import en   from '@/locales/en'
import ja   from '@/locales/ja'

function getSavedLocale() {
  try { return localStorage.getItem('app-locale') || 'zh-TW' } catch { return 'zh-TW' }
}

export const i18n = createI18n({
  legacy: false,              // 啟用 Composition API 模式
  locale: getSavedLocale(),   // 啟動時讀取已儲存的語系
  fallbackLocale: 'zh-TW',
  messages: { 'zh-TW': zhTW, en, ja },
})
```

在 `main.js` 掛載：

```js
import { i18n } from './i18n'
app.use(i18n)
```

### 13.4 語系狀態管理 — `src/stores/locale.js`

```js
import { defineStore } from 'pinia'
import { ref } from 'vue'
import { i18n } from '@/i18n'

export const useLocaleStore = defineStore('locale', () => {
  const locale = ref(localStorage.getItem('app-locale') || 'zh-TW')

  function setLocale(lang) {
    locale.value = lang
    i18n.global.locale.value = lang          // 更新 vue-i18n 當前語系
    localStorage.setItem('app-locale', lang) // 持久化語系偏好
    document.documentElement.lang = lang     // 更新 <html lang> 屬性
  }

  // Store 建立時同步初始值
  i18n.global.locale.value = locale.value
  document.documentElement.lang = locale.value

  return { locale, setLocale }
})
```

**語系切換流程：**

```
使用者點選語系選單
      ↓
localeStore.setLocale('en')
      ↓
① i18n.global.locale.value = 'en'   → vue-i18n 立即切換，所有 $t() 重新渲染
② localStorage.setItem('app-locale', 'en')  → 下次開啟頁面維持 English
③ document.documentElement.lang = 'en'      → HTML lang 屬性更新（SEO/無障礙）
```

### 13.7 語系持久化機制

```
localStorage key: app-locale
值範例: "zh-TW" | "en" | "ja"

讀取順序（頁面載入）：
1. i18n/index.js → getSavedLocale() 讀取 localStorage
2. i18n 以該語系初始化
3. locale store 建立時 → 同步 i18n.global.locale 與 html.lang

寫入（切換語系時）：
locale store.setLocale() → 同步更新以上三處
```

> **注意**：`locale` store 不使用 `pinia-plugin-persistedstate`，而是直接操作 `localStorage`，避免 Store 初始化與 i18n 初始化的時序問題（i18n 需在 Pinia 之前讀取初始語系）。

### 13.8 新增語系說明

若要新增一種語系（例如韓文 `ko`）：

1. 建立 `src/locales/ko.js`，照 `zh-TW.js` 結構翻譯所有 key
2. 在 `src/i18n/index.js` 引入並加入 `messages`：
   ```js
   import ko from '@/locales/ko'
   messages: { 'zh-TW': zhTW, en, ja, ko }
   ```
3. 在 `FrontLayout.vue`、`AdminLayout.vue`、`LoginView.vue` 的 `langs` 陣列新增：
   ```js
   { code: 'ko', flag: '🇰🇷', label: '한국어', short: 'KR' }
   ```

不需要修改 vue-i18n 設定或路由，新語系立即生效。
